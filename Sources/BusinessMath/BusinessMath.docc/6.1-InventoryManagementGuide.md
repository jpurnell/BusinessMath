# Inventory Management Models

Use analytical and simulation-based models to determine when and how much to order.

## Overview

The Operations module provides a suite of inventory management models drawn from standard operations management theory. These models answer two fundamental questions every business faces:

1. **When should I reorder?** — Safety stock and reorder point models
2. **How much should I order?** — EOQ and newsvendor models

Each model targets a specific business scenario, from stable-demand replenishment to single-period perishable goods. The ``InventoryAdvisor`` helps you choose the right model based on your available data.

### The Models at a Glance

| Model | Best For | Key Input |
|---|---|---|
| ``SafetyStockModel`` | Buffer against demand/lead time uncertainty | Service level target |
| ``ReorderPointModel`` | Continuous replenishment trigger | Demand history + lead time |
| ``EOQModel`` | Optimal order size | Ordering + holding costs |
| ``NewsvendorModel`` | Single-period / perishable items | Underage + overage costs |
| ``InventorySimulator`` | Non-normal demand, complex scenarios | Historical demand data |

## Safety Stock: Protecting Against Uncertainty

Safety stock is the buffer inventory held to protect against variability in demand and lead time. The ``SafetyStockModel`` provides three calculation methods:

### Demand-Only Method

When only demand varies and lead time is fixed:

`SS = z × σ_d × √L`

```swift
let ss = try SafetyStockModel<Double>.safetyStock(
    method: .demandOnly,
    serviceLevel: 0.95,
    averageDemand: 100.0,
    demandStdDev: 15.0,
    leadTime: 7.0
)
// ss ≈ 65.3 units
```

### Demand and Lead Time Method

When both demand and lead time are variable:

`SS = z × √(L × σ_d² + d̄² × σ_L²)`

```swift
let ss = try SafetyStockModel<Double>.safetyStock(
    method: .demandAndLeadTime,
    serviceLevel: 0.95,
    averageDemand: 100.0,
    demandStdDev: 15.0,
    leadTime: 7.0,
    leadTimeStdDev: 2.0
)
```

### Forecast Error Method

When you have a demand forecast with known accuracy:

`SS = z × RMSE × √L`

```swift
let ss = try SafetyStockModel<Double>.safetyStock(
    method: .forecastError,
    serviceLevel: 0.95,
    averageDemand: 100.0,
    demandStdDev: 15.0,
    leadTime: 7.0,
    forecastRMSE: 12.0
)
```

## Reorder Point: When to Place an Order

The ``ReorderPointModel`` combines expected demand during lead time with safety stock to determine the inventory level that triggers a replenishment order:

`r = d̄ × L + SS`

```swift
let result = try ReorderPointModel<Double>.calculate(
    demandHistory: dailySales,
    leadTime: 7.0,
    serviceLevel: 0.95
)
print("Reorder at \(result.reorderPoint) units")
print("Safety stock: \(result.safetyStock)")
print("Average daily demand: \(result.averageDailyDemand)")
```

### Stockout Probability

You can also estimate the probability of stocking out given your current inventory:

```swift
let prob = ReorderPointModel<Double>.stockoutProbability(
    currentStock: 500.0,
    averageDemand: 80.0,
    demandStdDev: 12.0,
    leadTime: 7.0
)
print("Stockout probability: \(prob * 100)%")
```

## EOQ: How Much to Order

The ``EOQModel`` uses the classical Harris-Wilson formula to find the order quantity that minimizes total annual inventory cost (ordering + holding):

`Q* = √(2SD / H)`

At the optimal quantity, annual ordering cost equals annual holding cost.

```swift
let result = try EOQModel<Double>.calculate(
    annualDemand: 10_000.0,
    orderingCost: 50.0,
    holdingCostPerUnit: 2.0
)
print("Order \(result.orderQuantity) units")
print("Total annual cost: $\(result.totalAnnualCost)")
print("Order every \(result.daysBetweenOrders) days")
```

## Newsvendor: Perishable and Seasonal Items

For products sold in a single period (perishables, seasonal goods, event merchandise), the ``NewsvendorModel`` balances the cost of ordering too many against ordering too few:

The **critical fractile** `p_c = c_u / (c_u + c_o)` determines the optimal service level, and the optimal quantity is `Q* = μ + z* × σ`.

```swift
let result = try NewsvendorModel<Double>.optimalQuantity(
    meanDemand: 100.0,
    demandStdDev: 25.0,
    underageCost: 5.0,   // lost profit per unit short
    overageCost: 2.0     // disposal cost per unit unsold
)
print("Order \(result.optimalQuantity) units")
print("Expected overstock: \(result.expectedOverstock)")
print("Expected understock: \(result.expectedUnderstock)")
```

## Monte Carlo Simulation

When demand is non-normal or you want to capture complex interactions, ``InventorySimulator`` runs thousands of demand-during-lead-time scenarios to build an empirical distribution:

```swift
let result = try InventorySimulator.simulate(
    demandHistory: last90DaysOfSales,
    meanLeadTime: 7.0,
    leadTimeStdDev: 2.0,
    serviceLevel: 0.95,
    strategy: .empirical,
    iterations: 10_000,
    seed: 42
)
print("Simulated reorder point: \(result.reorderPoint)")
print("Simulated safety stock: \(result.safetyStock)")
```

### Sampling Strategies

- **Empirical** (`.empirical`): Bootstraps from actual historical demand. Preserves the real distribution shape — good when you have ≥30 observations and suspect demand is not bell-shaped.
- **Normal** (`.normal`): Fits a Normal(μ, σ) to the demand history and samples from it. Appropriate when demand is approximately symmetric.

## Choosing the Right Model

Use ``InventoryAdvisor`` to get a data-driven recommendation:

```swift
let profile = InventoryAdvisor.DataProfile(
    demandHistory: dailySales,
    leadTimeMean: 7.0,
    leadTimeStdDev: 2.0,
    isPerishable: false
)
let rec = InventoryAdvisor.recommended(for: profile)
print(rec.recommendedModel)        // .reorderPoint
print(rec.safetyStockMethod!)      // .demandAndLeadTime
print(rec.simulationRecommended)   // true (if ≥30 observations)
for reason in rec.reasoning {
    print("• \(reason)")
}
```
